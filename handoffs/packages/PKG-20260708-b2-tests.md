## Scope

### In scope — case groups (one `it` each unless noted)

`blockActions.test.ts` — SET_BLOCK_TIME, unbudgeted (`estimatedHours: null`) tasks:
1. Grow within same-task same-person bin hours succeeds; bin row drained by the delta (deleted when fully consumed); activity message contains `pobrano z zasobnika`.
2. Grow past bin hours rejected (`toBe(state)`).
3. Grow with a bin row belonging to a DIFFERENT task or DIFFERENT person rejected (two cases — no cross-draw).
4. Move to another day/time with unchanged hours never rejected (no bin, no estimate).
5. Shrink still returns freed hours to the (created or merged) bin row.

`blockActions.test.ts` — INSERT_BLOCK budget:
6. Budgeted task with headroom only (no bin): insert within headroom succeeds (no bin suffix in the activity message), insert past headroom rejected.
7. Task with bin + headroom: insert draws bin-first (bin row decremented, remainder from headroom), activity message contains `pobrano z zasobnika`.
8. Insert exactly equal to the full allowance succeeds; allowance + 0.25 rejected.
9. Unbudgeted task: insert ≤ bin succeeds and drains the bin row; insert with zero bin rejected.
10. Insert with the picker's task DIFFERENT from the ref block's task uses the (selected task, ref person) allowance.

`blockActions.test.ts` — impersonation reducer:
11. `IMPERSONATE`: sets `currentUserId` to the target and `impersonatorId` to the previous user; no-op for a non-existent person and for `personId === currentUserId`.
12. Chained impersonation preserves the ORIGINAL impersonator; `IMPERSONATE` with `personId === impersonatorId` acts as return (both fields restored/cleared).
13. `STOP_IMPERSONATION`: restores `currentUserId` from `impersonatorId` and clears it; no-op when `''`.
14. `SET_CURRENT_USER` and `LOGOUT` both clear `impersonatorId` (LOGOUT also clears `currentUserId`).
15. `DELETE_PERSON` of the impersonated person returns the session to the impersonator; deleting the impersonator clears `impersonatorId` only. (Fixtures must satisfy the existing last-admin guard — keep an untouched administrator in `people`.)

`selectors.test.ts`:
16. `taskGrowAllowance`: null estimate ⇒ bin hours only; estimate set ⇒ bin + headroom; over-planned task (total > estimate) ⇒ headroom floored at 0 (bin still counts); no entries at all ⇒ 0.
17. `growAllowanceHours` returns a NUMBER for a null-estimate task's entry (equals that person's same-task bin hours) and 0 for a missing entry id.
18. `realUserId` / `realUser` / `isImpersonating`: not impersonating ⇒ self / false; impersonating ⇒ impersonator id / true.

`storage.test.ts`:
19. A persisted payload WITHOUT `impersonatorId` loads with `''` (default), and a valid non-empty `impersonatorId` round-trips.
20. Sanitize: `impersonatorId` referencing a non-existent person → `''`; `impersonatorId === currentUserId` → `''`; loading is idempotent (load → save-shape → load unchanged).
