2. **P2 — Project save can leave the form permanently dirty**
   **File:** [ProjectDetailPage.tsx](/Users/kacpercichyn2/Documents/N2click/src/pages/ProjectDetailPage.tsx:99)
   **Issue:** `save()` trims `name` and `description` before dispatch, but local state keeps the untrimmed values. Saving `"Foo "` stores `"Foo"`, then `dirty` remains true, so users still see unsaved-change prompts after a successful save.
   **Suggested fix:** Normalize local state on save with `setName(trimmed)` and `setDescription(description.trim())`, or compute dirty against the same normalized values that are persisted.
