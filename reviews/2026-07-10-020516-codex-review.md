   **File/area:** [src/store/AppStore.tsx](/Users/kacpercichyn2/Documents/N2click/src/store/AppStore.tsx:1559), [src/store/storage.ts](/Users/kacpercichyn2/Documents/N2click/src/store/storage.ts:605)  
   **Issue:** Deleting a work category clears `task.workCategoryId`, but saved task filter presets can still keep `criteria.workCategoryId` pointing at the deleted category. Applying that preset later sets an unselectable category filter and returns an empty task list with a `Kategoria: —` chip. `normalizeTaskMeta` also validates invalid `criteria.priority` but not dangling `criteria.workCategoryId`, so reload will not repair it.  
   **Suggested fix:** In `DELETE_WORK_CATEGORY`, also clear matching `savedFilters[].criteria.workCategoryId`. In `normalizeTaskMeta`, reset non-string or non-existent `criteria.workCategoryId` to `''`. Add tests for both reducer delete and load-time normalization.

2. **Severity: P3**  
   **File/area:** [src/store/AppStore.tsx](/Users/kacpercichyn2/Documents/N2click/src/store/AppStore.tsx:272) and [src/store/AppStore.tsx](/Users/kacpercichyn2/Documents/N2click/src/store/AppStore.tsx:294)  
   **Issue:** `SAVE_TASK` writes `draft.workCategoryId` without checking it against `state.workCategories`. A stale modal/imported action can persist a dangling category id until the next load normalizes it away, causing disappearing labels and inconsistent filtering.  
   **Suggested fix:** Validate `draft.workCategoryId` inside `saveTask` and write `''` unless it exists in `state.workCategories`. Add a reducer test for stale category ids on create/edit.
