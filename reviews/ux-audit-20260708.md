## P1 — broken function

1. **Calendar right-click insert can create same-person overlaps near end of day.**
   `INSERT_BLOCK` clamps inserted blocks back into the day bounds without running
   the same-person collision guard that normal drag/resize (`SET_BLOCK_TIME`) uses.
   Repro: insert "Dodaj po" after a block that ends near 24:00 so the clamp shifts
   the new block back onto existing blocks.
2. **Archiving an in-use status makes projects disappear from Kanban/pipeline views.**
   Kanban renders only active statuses, but Admin allows archiving a status that is
   still referenced by projects/tasks — those projects vanish from the board with no
   indication.

## P2 — friction

3. Kanban, Timeline, and Calendar scheduling are pointer-only; keyboard users can
   open items but not move/resize/reorder them. (Known accepted gap per CLAUDE.md,
   listed for completeness.)
4. Invalid calendar drops silently snap back after a temporary red tint, with no
   persistent explanation of why the drop failed.
5. Timeline/task period changes can remove scheduled work outside the new period
   without a strong confirmation/undo path.
6. Collapsed sidebar nav icons shrink to fit-content rather than uniform circular
   targets. (Already queued as Stream D of the current run.)
7. Mobile week calendar likely leaves too little usable room because the fixed bin
   pane remains visible.
8. `localStorage` save failures are swallowed, so users can lose work without warning.
9. Task validation blocks bad saves but does not focus the first invalid field, and
   numeric values are not all explicitly validated in app code.

## P3 — polish

10. Collapsed sidebar hides the visible search trigger even though `Ctrl/Cmd+K`
    still works.
11. Admin inline edits persist every keystroke without explicit save/cancel or undo.
12. Some warning/milestone/person indicators rely on symbols or color without
    consistently strong accessible text.
13. Saving a filter preset with an existing name overwrites it without warning.
