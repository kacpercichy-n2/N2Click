# Make local activity attribution honest

## Risk and routing
- Risk: high
- Route: architect → developer → reviewer
- Codex review: required

## Wiki context
- `openwiki/n2hub/state-and-persistence.md`
- `openwiki/n2hub/ui-navigation-and-onboarding.md`

## Expected touchpoints
- `src/store/AppStore.tsx`
- `src/store/selectors.ts`
- `src/store/permissions.ts`
- `src/types.ts`
- `src/store/*.test.ts`

## Invariants
- Local activity remains explicitly mutable and non-security-grade.
- Impersonated events retain both acting and real administrator identities.
- Existing authorization policy and storage conflict behavior do not change.

## Scope
- Add dual identity attribution for activity created while impersonating.
- Add non-duplicated local activity for role/person changes, status administration, deletions, password changes, login/logout and impersonation start/end where current reducer actions support them.
- Add focused reducer tests.

## Out of scope
- Backend audit storage, authorization-policy changes, reducer validation and navigation guards.

## Acceptance
- Impersonated entries identify both identities.
- Each covered sensitive action creates one clear activity entry.
- UI/docs do not imply that local activity is a security audit trail.

## Verification
- Worker: run the focused reducer/activity and permissions tests while iterating.
- Browser: none unless an existing login/impersonation interaction must change.
- Scheduler: owns the single final `npm test && npm run build` gate.
