MVP bundle: task metadata foundation.

Add task priority, work category, and lightweight task checklists.

Scope:
- Add `priority` to tasks with a small fixed set appropriate for the current UI, e.g. low / normal / high / urgent. Use Polish labels in the UI.
- Add a work category field. Prefer a small admin-managed dictionary if the current admin patterns make that straightforward; otherwise use a conservative enum and leave a clear extension point.
- Add task checklist items as part of the task model, not as separate tasks.
- UI:
  - `TaskModal`: edit priority, category, checklist.
  - `TasksPage`: show priority/category and filter them through `FilterPanel`.
  - Calendar/task cards may show compact priority/category affordances if it fits the existing design.
- Add storage migration with safe defaults for existing tasks.

Acceptance:
- Existing data migrates without loss.
- Checklist add/toggle/delete works inside `TaskModal`.
- Filters work and can be cleared.
- Add focused reducer/storage tests for migration and checklist persistence.
- Run `npm test` and `npm run build`.
- Update `CLAUDE.md` or handoff docs.
- Commit only this bundle to the review branch with a clear message.
