MVP bundle: PM operational dashboard.

Create or extend a PM-focused operational dashboard.

Scope:
- Either extend `DashboardPage` with role-aware PM sections or add `/pm-dashboard` if that fits the current navigation better.
- PM dashboard sections:
  - overdue tasks
  - unplanned tasks
  - partially planned tasks
  - blocked tasks
  - missing required documents/requirements if those models exist
  - estimate overruns
  - capacity/overload warnings
- Use existing selectors where possible; add pure selectors for new derived lists.
- Keep worker dashboard and employee `/my-work` behavior separate if already implemented.

Acceptance:
- PM/admin can scan operational problems from one page.
- Filters or links lead to the relevant task/project/client detail.
- Regular employees do not see restricted budget/formal data.
- Add selector tests for dashboard buckets.
- Run `npm test` and `npm run build`.
- Update docs/handoff notes.
- Commit only this bundle to the review branch with a clear message.
