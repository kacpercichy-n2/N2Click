MVP bundle: task execution controls.

Add actual time tracking, blocker reporting, and estimate change requests without mixing actual time with planned workload.

Scope:
- Actual time:
  - Add a separate data structure for real time, e.g. actual minutes per task/person/date.
  - Do not store actual time inside `workload`; workload remains the plan.
  - Show `estymacja / zaplanowano / faktycznie` where task effort is shown.
- Blockers:
  - Add blocker data, either fields like `blockedReason`, `blockedBy`, `blockedAt` or a small `TaskBlocker` entity if that fits better.
  - UI in `TaskModal`: report/clear blocker.
  - Show blocked state as a red alert/badge and expose a filter.
- Estimate change request:
  - Add `EstimateChangeRequest`: current estimate, proposed estimate, reason, status.
  - PM/admin can accept/reject from the task or dashboard.
  - Acceptance updates the task estimate and logs activity.

Acceptance:
- Planned workload and actual time stay separate in selectors and UI.
- Blocked tasks are visible and filterable.
- Estimate request lifecycle is clear: pending, accepted, rejected.
- Add migration and focused tests for actual time/blockers/estimate requests.
- Run `npm test` and `npm run build`.
- Update docs/handoff notes.
- Commit only this bundle to the review branch with a clear message.
