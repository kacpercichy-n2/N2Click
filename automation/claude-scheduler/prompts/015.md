MVP bundle: documents and formal readiness.

Add project/client document links, document categories, project requirements, planning block, and override history.

Scope:
- Add document data model:
  - type/scope: client or project
  - name
  - URL/file reference field
  - category
  - verified flag
- Add document categories: umowy, oferty, briefy, faktury, materiały, prawne, inne.
- Add `ProjectRequirement`: name, required flag, fulfilled flag, linked document id if applicable.
- UI:
  - `ProjectDetailPage`: documents section and requirements checklist.
  - Admin/category management only if it follows existing patterns; otherwise keep fixed categories for this bundle.
- Formal readiness:
  - Before planning actions such as `SET_BLOCK_TIME`, `SAVE_TASK`, or dragging from bin, check whether required project requirements are fulfilled.
  - PM/admin can override with a reason.
  - Log override in existing activity history.

Acceptance:
- Planning block prevents normal users from scheduling work on formally unready projects.
- PM/admin override requires a reason and appends an activity event.
- Documents and requirements persist through reload.
- Add reducer/storage tests for requirements, override activity, and blocked planning.
- Run `npm test` and `npm run build`.
- Update docs/handoff notes.
- Commit only this bundle to the review branch with a clear message.
