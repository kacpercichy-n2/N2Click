# Add one deterministic release browser command

## Risk and routing
- Risk: medium
- Route: developer → reviewer
- Codex review: conditional — run only if orchestration changes scenario semantics or reviewer uncertainty remains

## Wiki context
- `openwiki/n2hub/testing-and-automation.md`

## Expected touchpoints
- `new: scripts/run-browser-regression.mjs`
- `package.json`
- `automation/claude-scheduler/README.md`

## Invariants
- Existing Chromium/WebKit scenario assertions and failure conditions remain intact.
- The runner starts and stops its own preview server deterministically.
- It does not rebuild individual browser scenarios.

## Scope
- Add one documented package command that performs the production build and existing critical browser regression scripts.
- Reuse existing scripts and make clean-install prerequisites explicit.

## Out of scope
- New product behavior, new browser scenarios and broad documentation refresh.

## Acceptance
- One command runs the documented release browser suite without a manually started dev server.
- Failure of any scenario produces a non-zero exit and cleans up the server.

## Verification
- Worker: smoke the orchestration with one representative Chromium scenario before the full command.
- Browser: the final new release command owns its declared Chromium/WebKit matrix once.
- Scheduler: owns the single final `npm test && npm run build` gate; do not repeat the release matrix there.
